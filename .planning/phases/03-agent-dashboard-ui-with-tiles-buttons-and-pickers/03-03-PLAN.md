<objective>
Fix workspace-scoped data storage bug that causes the sidebar to lose all agents and repos when focusAgent switches workspace folders.

Purpose: All three services (AgentService, RepoConfigService, WorktreeService) currently receive `context.workspaceState` which is scoped to the current workspace folder. When focusAgent calls `updateWorkspaceFolders` to switch to an agent's worktree, VS Code re-activates the extension with a new, empty `workspaceState` -- wiping all data. Switching to `context.globalState` makes data persist across workspace folder changes.

Output: Extension stores all persistent data in globalState, sidebar retains all agents/repos after workspace switching.
</objective>
