<objective>
Implement the core git infrastructure: an async GitService wrapper (enforcing PERF-04), a worktree parser utility, and a WorktreeService that creates/removes worktrees with hard limits (GIT-05), tracks them in a Memento-backed manifest (GIT-06), and reconciles manifest vs disk state on demand. This is the most architecturally significant code in Phase 1.

Purpose: Git worktree isolation is the core primitive of the entire extension. Every agent gets its own worktree. This plan builds the foundation that Phases 2-7 depend on.
Output: Three production files (GitService, WorktreeService, parseWorktreeList) with comprehensive unit tests.
</objective>

<task type="auto" tdd="true">
  <name>Task 2: WorktreeService -- CRUD, hard limits, manifest, reconciliation</name>
  <files>src/services/worktree.service.ts, test/unit/worktree.service.test.ts</files>
  <behavior>
    WorktreeService:
    - constructor(gitService, workspaceState) -- receives dependencies via constructor injection
    - addWorktree(repoPath, agentName, startPoint?) creates worktree at repoPath/.worktrees/agentName/, creates branch, adds to manifest, returns WorktreeEntry
    - addWorktree throws when worktree count for repo >= worktreeLimit (GIT-05 hard limit enforcement)
    - removeWorktree(repoPath, agentName) runs `git worktree remove`, deletes branch, removes from manifest
    - removeWorktree handles case where worktree already missing from disk gracefully
    - getManifest(repoPath) returns WorktreeEntry[] for that repo from workspaceState
    - reconcile(repoPath) compares manifest vs `git worktree list --porcelain` output, returns { orphanedInManifest, orphanedOnDisk, healthy }
    - reconcile removes orphanedInManifest entries from manifest
    - reconcile removes orphanedOnDisk worktrees from disk (only those under .worktrees/)
    - Uses a per-repo mutex to prevent TOCTOU races on concurrent addWorktree calls
  </behavior>
  <action>
1. Create src/services/worktree.service.ts:
   - Import types from src/models/worktree.ts and src/models/repo.ts
   - Import parseWorktreeList from src/utils/worktree-parser.ts
   - Import path from 'node:path'
   - Import vscode Memento type

   WorktreeService class:
   - constructor(private readonly git: GitService, private readonly state: vscode.Memento)
   - private locks: Map<string, Promise<void>> for per-repo mutex

   - private async withLock(repoPath: string, fn: () => Promise<T>): Promise<T> -- simple mutex using a promise chain per repo path. Queues operations so only one addWorktree/removeWorktree runs at a time per repo.

   - async addWorktree(repoPath: string, agentName: string, startPoint?: string): Promise<WorktreeEntry>
     - Acquire lock for repoPath
     - Read current manifest for this repo
     - Check count against limit (from RepoConfig -- accept limit as parameter or read from state). If count >= limit, throw Error with descriptive message: "Worktree limit ({limit}) reached for {repoPath}. Delete an existing agent to make room."
     - Compute worktree path: path.join(repoPath, '.worktrees', agentName)
     - Branch name: agentName (agents get their own branch)
     - Run git.exec(repoPath, ['worktree', 'add', '-b', branchName, worktreePath, startPoint || 'HEAD'])
     - Create WorktreeEntry with path, branch, agentName, repoPath, createdAt: new Date().toISOString()
     - Append to manifest and save
     - Return entry

   - async removeWorktree(repoPath: string, agentName: string): Promise<void>
     - Acquire lock for repoPath
     - Find entry in manifest by agentName and repoPath
     - Try git.exec(repoPath, ['worktree', 'remove', '--force', entry.path]) -- catch and log if worktree already gone
     - Try git.exec(repoPath, ['branch', '-D', entry.branch]) -- catch and log if branch already gone
     - Remove from manifest and save

   - getManifest(repoPath: string): WorktreeEntry[]
     - Read all entries from state, filter by repoPath

   - private getAllManifestEntries(): WorktreeEntry[]
     - return this.state.get<WorktreeEntry[]>(WORKTREE_MANIFEST_KEY, [])

   - private async saveManifest(entries: WorktreeEntry[]): Promise<void>
     - await this.state.update(WORKTREE_MANIFEST_KEY, entries)

   - async reconcile(repoPath: string): Promise<ReconciliationResult>
     - Get manifest entries for this repo
     - Run git.exec(repoPath, ['worktree', 'list', '--porcelain']) and parse with parseWorktreeList
     - Compare: entries in manifest but not on disk = orphanedInManifest; entries on disk (under .worktrees/) but not in manifest = orphanedOnDisk; entries in both = healthy
     - For orphanedInManifest: remove from manifest
     - For orphanedOnDisk: run git worktree remove --force
     - Save updated manifest
     - Return result

   Export ReconciliationResult interface: { orphanedInManifest: WorktreeEntry[], orphanedOnDisk: WorktreeOnDisk[], healthy: WorktreeEntry[] }
