  <action>
    **1. Create `src/commands/diff.commands.ts`:**

    Export `registerDiffCommands(context, diffService, repoConfigService, agentService)` function that registers two commands:

    **`vscode-agentic.reviewChanges` command (args: repoPath, agentName):**
    1. Call `diffService.getChangedFiles(repoPath, agentName)` to get file list
    2. Get staging branch name via `repoConfigService.getForRepo(repoPath)?.stagingBranch`
    3. If no changed files, show info message "No changes between {agentName} and {staging}" and return
    4. Show QuickPick with items: `label = filename (last segment)`, `description = full relative path`, carry `_filePath` metadata
    5. PlaceHolder: `{N} file(s) changed vs {staging}`, Title: `Changes: {agentName} vs {staging}`
    6. On selection: build left URI via `GitContentProvider.buildUri(repoPath, staging, filePath)` and right URI via `GitContentProvider.buildUri(repoPath, agentName, filePath)`
    7. Execute `vscode.commands.executeCommand("vscode.diff", leftUri, rightUri, "{filename} ({staging} <-> {agentName})")`

    **`vscode-agentic.createPR` command (args: repoPath, agentName):**
    1. Get staging branch and changed files count
    2. Show confirmation via `showInformationMessage` with details: "Create PR: {agentName} -> {staging} ({N} changed files)?", buttons: "Create PR" / "Cancel"
    3. If confirmed: `execFileAsync("gh", ["pr", "create", "--base", staging, "--head", agentName, "--title", "Agent: {agentName}", "--body", "Changes from agent {agentName}"], { cwd: repoPath, timeout: 30_000 })`
    4. On success: `showInformationMessage("PR created: {stdout.trim()}")` (gh outputs the PR URL)
    5. On ENOENT error: `showErrorMessage("GitHub CLI (gh) is not installed. Install from https://cli.github.com/")`
    6. On other error (auth, etc.): `showErrorMessage("Failed to create PR: {error.stderr || error.message}")`

    **2. Modify `src/commands/agent.commands.ts`:**
    - Add `DiffService` parameter to `registerAgentCommands` signature
    - In the `deleteAgent` command handler, BEFORE the confirmation dialog, add merge guard:
      1. After user selects an agent from QuickPick, call `diffService.hasUnmergedChanges(repoPath, agentName)`
      2. If true: show `showWarningMessage("Agent '{agentName}' has unmerged changes vs staging. Review changes or create a PR first.", "Review Changes", "Cancel")`
      3. If user clicks "Review Changes": `executeCommand("vscode-agentic.reviewChanges", repoPath, agentName)`
      4. Return without deleting (hard block)
      5. If false: proceed with existing confirmation+deletion flow

    **3. Modify `src/commands/sidebar.commands.ts`:**

      2. If true: show same warning with "Review Changes" / "Cancel" buttons

      4. Return without deleting
      5. If false: proceed with existing flow

    - Add private `diffStatusCache = new Map<string, boolean>()` for async diff status
    - In `getChildren` for repo group level: pass `this.diffStatusCache.get(`${a.repoPath}::${a.agentName}`) ?? false` as `hasDiffs` to AgentTreeItem constructor
    - Add `async updateDiffStatus(): Promise<void>` method:
      1. Get all agents via `agentService.getAll()`
      2. For each agent, call `diffService.hasUnmergedChanges(agent.repoPath, agent.agentName)`
      3. Update `diffStatusCache` with results
      4. Call `this.refresh()` to trigger tree re-render with new contextValues
    - In the `agentChangeSubscription` callback (constructor): after debounced refresh, also call `this.updateDiffStatus()` (debounced separately or chained after refresh)
    - Important: Do NOT await diff checks in getChildren -- read from cache only (per research pitfall #3). The async updateDiffStatus runs separately and triggers a second refresh when results arrive.
