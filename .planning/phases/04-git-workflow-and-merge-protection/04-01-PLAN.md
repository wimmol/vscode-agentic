<task type="auto" tdd="true">
  <name>Task 1: DiffService with git diff detection and changed file listing</name>
  <files>src/services/diff.service.ts, test/unit/diff.service.test.ts</files>
  <behavior>
    - hasUnmergedChanges returns false when repoConfig.getForRepo returns undefined (no config)
    - hasUnmergedChanges returns false when staging branch does not exist (branchExists returns false)
    - hasUnmergedChanges returns false when git diff --name-only returns empty output
    - hasUnmergedChanges returns true when git diff --name-only returns file names
    - hasUnmergedChanges returns false when git diff throws an error (graceful degradation)
    - getChangedFiles returns empty array when repoConfig returns undefined
    - getChangedFiles returns empty array when git diff throws
    - getChangedFiles returns array of file paths parsed from newline-separated git output
    - getChangedFiles filters out empty strings from split output
    - Both methods use three-dot notation: `staging...agentBranch`
  </behavior>
  <action>
    Create `src/services/diff.service.ts` with class `DiffService`:
    - Constructor takes `GitService` and `RepoConfigService` (constructor injection per project pattern)
    - `async hasUnmergedChanges(repoPath: string, agentBranch: string): Promise<boolean>`:
      1. Get RepoConfig via `repoConfig.getForRepo(repoPath)` -- return false if undefined
      2. Check `git.branchExists(repoPath, config.stagingBranch)` -- return false if staging doesn't exist
      3. Run `git.exec(repoPath, ["diff", "--name-only", `${staging}...${agentBranch}`])`
      4. Return `output.trim().length > 0`
      5. Catch errors and return false (graceful degradation per research pitfall #1)
    - `async getChangedFiles(repoPath: string, agentBranch: string): Promise<string[]>`:
      1. Get RepoConfig -- return [] if undefined
      2. Run same three-dot diff command
      3. Return `output.trim().split("\n").filter(Boolean)`
      4. Catch errors and return []

    Test file follows existing patterns (see test/unit/git.service.test.ts). Mock GitService and RepoConfigService with vi.fn(). Test both happy paths and error paths.
  </action>
  <verify>
    <automated>npx vitest run test/unit/diff.service.test.ts --reporter=verbose</automated>
  </verify>
  <done>DiffService has hasUnmergedChanges and getChangedFiles methods. All unit tests pass covering: no config, no staging branch, empty diff, non-empty diff, and git error cases.</done>
</task>

    - Import vscode and GitService type
    - Class `GitContentProvider` implements `vscode.TextDocumentContentProvider`
    - Static `readonly SCHEME = "agentic-git"`
    - Constructor takes `GitService`
    - `async provideTextDocumentContent(uri: vscode.Uri): Promise<string>`:
      1. Parse query params via `new URLSearchParams(uri.query)`
      2. Extract `repo`, `ref`, `path` params

      4. Return result; catch errors and return "" (file doesn't exist at ref)
    - Static `buildUri(repoPath: string, ref: string, filePath: string): vscode.Uri`:
      1. Build URI with scheme, path as `/${filePath}`, query with encoded repo, ref, path params

    Note on URI construction: Use `encodeURIComponent` for all query parameter values to handle paths with spaces and special characters (research pitfall #6).
