    2. Create src/services/workspace-switch.service.ts:
       - Export `WorkspaceSwitchService` class:
         - Constructor takes AgentService, TerminalService, WorktreeService
         - Private `activeAgent: { repoPath: string; agentName: string } | undefined`
         - Public `getActiveAgent()` returns the active agent tuple or undefined
         - Public async `switchToAgent(repoPath: string, agentName: string): Promise<void>`:
           a. Determine if same-repo: `this.activeAgent?.repoPath === repoPath`
           b. Always call `agentService.focusAgent(repoPath, agentName)` -- handles terminal show/create
           c. If cross-repo (different repoPath or no active agent):
              - Get worktree path from `worktreeService.getManifest(repoPath).find(w => w.agentName === agentName)?.path`
              - If worktreePath found:
                - Check `vscode.workspace.workspaceFolders` for existing folder match
                - If not found, call `vscode.workspace.updateWorkspaceFolders(folders.length, 0, { uri: vscode.Uri.file(worktreePath) })`
                - Focus file explorer: `await vscode.commands.executeCommand("workbench.view.explorer")`
                - Reveal in explorer: `await vscode.commands.executeCommand("revealInExplorer", vscode.Uri.file(worktreePath))`
           d. Update `this.activeAgent = { repoPath, agentName }`

    3. Create src/commands/sidebar.commands.ts:
       - Export `registerSidebarCommands(context, agentService, workspaceSwitchService, treeView, treeProvider)`:
       - Register `vscode-agentic.focusAgentFromTile`:
         - Called by TreeItem.command with arguments [repoPath, agentName]
         - Calls `workspaceSwitchService.switchToAgent(repoPath, agentName)`
         - After switch, reveal the agent in the TreeView for active highlighting:
           Create a new AgentTreeItem(agentName, repoPath, "created") as reveal target (TreeView matches by id, status does not matter for id).
           Call `treeView.reveal(agentItem, { select: true, focus: false })`
       - Register `vscode-agentic.deleteAgentFromTile`:
         - Receives (repoPath, agentName) as arguments from context menu
         - Show warning dialog: "Delete agent '{agentName}'? This removes the worktree and branch."
         - If user confirms "Delete", call `agentService.deleteAgent(repoPath, agentName)`
         - Show info message on success
       - Register `vscode-agentic.copyBranchName`:
         - Receives (repoPath, agentName) as arguments from context menu
         - Call `vscode.env.clipboard.writeText(agentName)` -- agentName IS the branch name (per Phase 1 design)
         - Show info message: "Copied branch name: {agentName}"
       - Register `vscode-agentic.createAgentInRepo`:
         - Receives repoPath as argument from inline button on repo group
         - Call `vscode.commands.executeCommand("vscode-agentic.createAgent", repoPath)` -- pass repoPath as arg to createAgent command
