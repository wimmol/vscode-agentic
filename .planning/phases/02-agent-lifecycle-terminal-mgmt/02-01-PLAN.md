  <name>Task 2: TerminalService with lifecycle management</name>
  <files>src/services/terminal.service.ts, test/unit/terminal.service.test.ts</files>
  <behavior>
    - createTerminal creates a VS Code terminal with name="Agent: {agentName}", shellPath="claude", cwd=worktreePath, isTransient=true
    - createTerminal passes initialPrompt as shellArgs[0] when provided; empty shellArgs when no prompt
    - createTerminal returns the existing terminal and calls show() if one already exists for the same agent
    - createTerminal stores the terminal in the internal map keyed by repoPath::agentName
    - disposeTerminal removes the map entry BEFORE calling terminal.dispose()
    - disposeTerminal is a no-op when no terminal exists for the agent
    - handleTerminalClose identifies the correct agent from the map by terminal identity (===)
    - handleTerminalClose calls onStatusChange with status="finished" when exitCode is 0 or undefined
    - handleTerminalClose calls onStatusChange with status="error" when exitCode is non-zero (e.g., 1, 127)
    - handleTerminalClose removes the terminal from the map
    - handleTerminalClose is a no-op for terminals not in the map (non-agent terminals)
    - Multiple terminals for different agents can coexist in the map
    - showTerminal calls show() on an existing terminal; is a no-op for unknown agents
    - hasTerminal returns true when a terminal exists for the agent, false otherwise
    - dispose() cleans up event subscriptions
  </behavior>
  <action>
    Create `src/services/terminal.service.ts`:

    ```typescript
    import * as vscode from "vscode";
    import type { AgentStatus } from "../models/agent.js";
    ```

    Implement `TerminalService` class:
    - Private field `terminals: Map<string, vscode.Terminal>` for agent-to-terminal mapping
    - Private field `disposables: vscode.Disposable[]` for event cleanup
    - Constructor takes `onStatusChange: (agentName: string, repoPath: string, status: AgentStatus, exitCode?: number) => void` callback
    - In constructor, subscribe to `vscode.window.onDidCloseTerminal` and store the disposable
    - Private method `terminalKey(repoPath: string, agentName: string): string` returns `${repoPath}::${agentName}`
    - Method `createTerminal(repoPath, agentName, worktreePath, initialPrompt?)`: If terminal exists for key, call `.show()` and return it. Otherwise create via `vscode.window.createTerminal({ name: "Agent: ${agentName}", shellPath: "claude", shellArgs: initialPrompt ? [initialPrompt] : [], cwd: worktreePath, isTransient: true })`, store in map, return terminal.
    - Method `disposeTerminal(repoPath, agentName)`: Get terminal from map. If found, DELETE from map FIRST, then call `terminal.dispose()`. Critical ordering to prevent race with close handler (per research Pitfall 2).
    - Method `showTerminal(repoPath, agentName)`: Get terminal from map, call `.show()` if found.
    - Method `hasTerminal(repoPath, agentName): boolean`: Return whether key exists in map.
    - Private method `handleTerminalClose(terminal: vscode.Terminal)`: Iterate map entries, find matching terminal by `===` identity comparison. If found, remove from map, extract repoPath and agentName from key, determine status (exitCode !== undefined && exitCode !== 0 ? "error" : "finished"), call `this.onStatusChange(agentName, repoPath, status, exitCode)`. If not found (non-agent terminal), do nothing.
    - Method `dispose()`: Dispose all disposables, clear array.
