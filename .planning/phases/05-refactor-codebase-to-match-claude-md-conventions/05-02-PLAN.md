2. Create src/ui/hooks/useVsCodeApi.ts:
   - Declare `acquireVsCodeApi` as a global function (it exists in webview context)
   - Call it exactly once at module level and cache the result
   - Export a `getVsCodeApi()` function that returns the cached instance
   - Export a `postCommand(command: string, data?: Record<string, string>)` helper that calls `vsCodeApi.postMessage({ command, ...data })`

3. Create src/ui/hooks/useElapsedTime.ts:
   - Function `useElapsedTime(startedAt: string | undefined, finishedAt: string | undefined, isRunning: boolean): string`
   - Uses useState for current time, useEffect with setInterval(1000) when isRunning
   - Returns formatted string: "--" if no startedAt, "Xs" if < 60s, "Xm Ys" if < 3600s, "Xh Ym" otherwise
   - Cleans up interval on unmount or when isRunning changes to false

4. Create src/ui/atoms/StatusIcon.tsx:
   - Props: `{ status: AgentData["status"] }`
   - Returns a `<span>` with appropriate codicon class:
     - running: `codicon codicon-loading spin`
     - created: `codicon codicon-person`
     - finished: `codicon codicon-check`
     - error: `codicon codicon-error`
   - Add appropriate CSS class for status color (e.g., `status-running`, `status-error`)

5. Create src/ui/atoms/ActionButton.tsx:
   - Props: `{ icon: string; title: string; disabled?: boolean; onClick: () => void }`
   - Renders a `<button>` with codicon icon, title attribute, disabled state
   - onClick only fires when not disabled
   - CSS class: `action-button`

6. Create src/ui/atoms/ElapsedTimer.tsx:
   - Props: `{ startedAt?: string; finishedAt?: string; isRunning: boolean }`
   - Uses the `useElapsedTime` hook
   - Renders formatted time in a `<span class="elapsed-time">`
