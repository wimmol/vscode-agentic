The workspace file (`~/.agentic/agentic.code-workspace`) is a NEW file this plan creates -- it reads repo paths FROM globalState via RepoConfigService and writes them TO the workspace file for VS Code workspace naming.

    Write tests FIRST in test/unit/workspace.service.test.ts, then implement src/services/workspace.service.ts.

    IMPORTANT (data source): The repo list comes from `repoConfigService.getAll()` which reads from VS Code globalState (Memento), NOT from `~/.agentic/config.json`. The config.json file does not exist in the current architecture. CONTEXT.md references config.json but this was a user mental model -- the actual implementation uses globalState per the established pattern from Phase 1. Do NOT attempt to read or create config.json.

    WorkspaceService class:
    - Constructor receives RepoConfigService (for getAll())
    - Private `workspaceDir` = `path.join(os.homedir(), ".agentic")`
    - Private `workspaceFilePath` = `path.join(workspaceDir, "agentic.code-workspace")`
    - `getWorkspaceFilePath(): string` -- returns the path
    - `isInWorkspaceMode(): boolean` -- checks `vscode.workspace.workspaceFile` is a file:// URI ending in "agentic.code-workspace"
    - `async ensureWorkspaceFile(): Promise<boolean>` -- creates/updates the .code-workspace file if repos exist. Returns true if file was created for first time (caller should prompt reopen). Uses `fs.mkdir` with `recursive: true` for directory, `fs.writeFile` for file. Reads existing file first to avoid unnecessary writes.
    - `async syncWorkspaceFile(): Promise<void>` -- writes workspace file with current repo list from repoConfigService.getAll(). Each folder entry has `path` (absolute) and `name` (basename).
    - `async promptReopenInWorkspace(): Promise<void>` -- shows notification per CONTEXT.md spec, executes `vscode.openFolder` with workspace file URI if user accepts.
    - `setExplorerScope(mode: "global" | {repo: string} | {repo: string, agent: string, worktreePath: string}): void` -- calls `vscode.workspace.updateWorkspaceFolders()` with the appropriate folders. For "global": all repo roots with basenames. For repo: single repo root. For agent: single worktree path (existing behavior).

    Use promise-chain mutex pattern (already established in WorktreeService) for file writes to prevent race conditions per Research Pitfall 2.

    Mock `node:fs/promises` in tests using `vi.mock("node:fs/promises")`. Mock `node:os` to return a fixed homedir. The vscode mock already has workspace, Uri, window, commands.
